# Story 02-04 二维码扫描查询

## 0. 基本信息

- Epic：库存查询
- Story ID：02-04
- 优先级：P1
- 状态：Draft
- 预计工期：0.5d
- 依赖 Story：01-04（物料档案新增）
- 关联迭代：Sprint 1

---

## 1. 目标

实现二维码扫描查询能力，支持扫码显示物料、批次、库位信息。

---

## 2. 业务背景

二维码扫描是仓库作业的核心交互方式，支持快速定位物料信息。

---

## 3. 范围

### 3.1 本 Story 包含

- 二维码扫码查询
- 显示物料、批次、库位信息
- 显示当前库存数量

### 3.2 本 Story 不包含

- 二维码绑定
- 二维码打印

---

## 4. 参与角色

- 仓库作业员：扫码查询物料信息

---

## 5. 前置条件

- 物料已绑定二维码
- 用户具备二维码查询权限

---

## 6. 触发方式

- 页面入口：库存管理 → 二维码查询
- 接口入口：GET `/api/inventory/qrcode/{qrcodeValue}`
- 硬件入口：PDA 扫码

---

## 7. 输入 / 输出

### 7.1 输入

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| qrcodeValue | string | Y | 二维码值（UDI编码） |

### 7.2 输出

| 字段 | 类型 | 说明 |
|---|---|
| qrcodeValue | string | 二维码值 |
| materialCode | string | 物料编码 |
| materialName | string | 物料名称 |
| spec | string | 规格 |
| unit | string | 单位 |
| batchNo | string | 批次号 |
| warehouseCode | string | 仓库编码 |
| warehouseName | string | 仓库名称 |
| locationCode | string | 库位编码 |
| locationName | string | 库位名称 |
| quantity | int | 当前库存数量 |
| produceDate | date | 生产日期 |
| expireDate | date | 失效日期 |

---

## 8. 页面/交互要求

### 8.1 页面

- 页面名称：二维码扫描页
- 页面类型：扫描+详情页
- 菜单路径：库存管理 → 二维码查询

### 8.2 交互要求

- 打开页面调起扫码
- 扫码后显示物料详情
- 点击【重新扫描】继续扫码

### 8.3 展示字段

| 字段 | 说明 |
|---|---|
| 物料编码 | - |
| 物料名称 | - |
| 规格 | - |
| 批次号 | - |
| 仓库 | - |
| 库位 | - |
| 库存数量 | - |
| 生产日期 | - |
| 失效日期 | 临期标黄 |

---

## 9. 业务规则

1. **扫码响应 < 200ms**
2. **二维码值唯一**
3. **数据权限**：仅显示用户有权限的仓库

---

## 10. 验收标准

### 10.1 功能验收

- [ ] 物料信息显示正确
- [ ] 库位信息显示正确
- [ ] 库存数量显示正确
