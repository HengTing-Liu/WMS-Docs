## 一、性能指标要求

### 1.1 响应时间要求

| 场景 | 指标 | 说明 |
|------|------|------|
| 页面首次加载 | ≤ 3秒 | 首屏渲染完成 |
| 简单查询（列表） | ≤ 1秒 | 100条以内数据 |
| 复杂查询 | ≤ 3秒 | 带聚合统计的查询 |
| 单条数据操作（新增/修改） | ≤ 500ms | 不含复杂业务逻辑 |
| 批量操作（100条以内） | ≤ 5秒 | 批量新增/修改/删除 |
| 导出Excel（1000条） | ≤ 10秒 | 含样式生成 |
| 二维码生成（单张） | ≤ 200ms | 含打印排版 |
| 文件上传（10MB以内） | ≤ 5秒 | 含病毒扫描 |

### 1.2 并发要求

| 场景 | 指标 | 说明 |
|------|------|------|
| 系统最大并发用户数 | ≥ 500 | 同时在线用户 |
| 系统吞吐量（TPS） | ≥ 100 | 每秒处理事务数 |
| API最大并发 | ≥ 200 | 单接口最大并发 |

### 1.3 可用性要求

| 场景 | 指标 | 说明 |
|------|------|------|
| 系统可用性 | ≥ 99.9% | 全年停机时间 ≤ 8.7小时 |
| 核心业务可用性 | ≥ 99.99% | 入库/出库等核心流程 |
| 故障恢复时间 | ≤ 30分钟 | 从故障到恢复 |
| 数据备份 | 每小时增量 | 每日全量 |

## 二、数据库性能要求

### 2.1 查询性能

| 场景 | 指标 | 说明 |
|------|------|------|
| 单表查询（无索引） | ≤ 500ms | 10万条数据 |
| 单表查询（有索引） | ≤ 100ms | 100万条数据 |
| JOIN查询（3表以内） | ≤ 1秒 | 含索引优化 |
| 分页查询（深度分页） | ≤ 500ms | 第100页 |

### 2.2 索引规范

1. **必须建立索引**：
   - 主键、外键
   - 常用查询条件字段
   - 排序字段
   - 唯一约束字段

2. **禁止全表扫描**：
   - 单次查询返回数据量不超过1000条
   - 使用LIMIT分页

3. **索引维护**：
   - 定期分析表统计信息
   - 监控慢查询日志
   - 定期清理无用索引

### 2.3 SQL规范

```sql
-- ✅ 正确示例
SELECT id, warehouse_code, warehouse_name
FROM wms_warehouse
WHERE is_enabled = 1
  AND warehouse_type = 'SALES'
ORDER BY created_at DESC
LIMIT 20 OFFSET 0;

-- ❌ 错误示例
SELECT * FROM wms_warehouse WHERE is_enabled = 1;
```

## 三、缓存设计

### 3.1 缓存策略

| 缓存类型 | 缓存时间 | 适用场景 |
|----------|----------|----------|
| 字典数据 | 24小时 | 字典类型、枚举值 |
| 用户信息 | 30分钟 | 用户基本信息 |
| 权限信息 | 30分钟 | 用户角色、菜单权限 |
| 仓库信息 | 1小时 | 仓库列表 |
| 库位信息 | 30分钟 | 库位树结构 |
| 物料信息 | 1小时 | 物料基础信息 |

### 3.2 缓存更新策略

1. **主动更新**：数据变更时主动删除缓存
2. **被动过期**：设置TTL自动过期
3. **异步刷新**：缓存即将过期时异步更新

### 3.3 缓存示例

```java
// Redis缓存注解
@Cacheable(value = "dict", key = "#dictType", expire = 86400)
public List<DictData> getDictDataByType(String dictType) {
    return dictMapper.selectByType(dictType);
}

// 数据变更时删除缓存
@CacheEvict(value = "dict", key = "#dictType")
public void updateDictData(String dictType, DictData data) {
    dictMapper.update(data);
}
```

## 四、异步处理

### 4.1 异步场景

| 场景 | 异步方式 | 说明 |
|------|----------|------|
| 发送通知 | MQ消息 | 邮件、短信、站内信 |
| 日志记录 | 异步日志 | ELK收集 |

| 报表生成 | 异步任务 | XXL-Job |
| 库存计算 | RocketMQ | 库存变更通知 |

### 4.2 消息队列配置

```yaml
rocketmq:
  producer:
    group: wms-producer-group
    send-message-timeout: 3000
    retry-times-when-send-failed: 2
  consumer:
    group: wms-consumer-group
    consume-thread-min: 10
    consume-thread-max: 50
```

## 六、性能优化建议

### 6.1 前端优化

1. **资源优化**：
   - 图片压缩、雪碧图
   - JS/CSS合并压缩
   - 懒加载非首屏资源

2. **接口优化**：
   - 分页加载
   - 虚拟滚动
   - 请求缓存

3. **渲染优化**：
   - 列表虚拟滚动
   - 减少重排重绘

### 6.2 后端优化

1. **数据库优化**：
   - 读写分离
   - 分库分表（后期）
   - 索引优化

2. **缓存优化**：
   - 多级缓存
   - 本地缓存 + 分布式缓存

3. **并发优化**：
   - 线程池调优
   - 异步处理

4. **架构优化**：
   - 服务拆分
   - 读写分离
   - 动静分离
