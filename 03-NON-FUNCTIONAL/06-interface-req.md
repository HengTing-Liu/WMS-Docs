## 六、容错处理

### 6.1 重试策略

| 参数 | 配置 |
|------|------|
| 最大重试次数 | 3次 |
| 重试间隔 | 1分钟、5分钟、15分钟 |
| 幂等键 | requestId |
| 死信队列 | DLQ |

### 6.2 降级策略

| 场景 | 降级方案 |
|------|----------|
| ERP不可用 | 本地缓存，延迟同步 |
| LIMS不可用 | 人工质检降级 |
| MQ不可用 | 本地队列重试 |
| 接口超时 | 超时重试 |

### 6.3 补偿机制

```java
@RocketMQListener(topic = "erp-purchase-order")
public void handlePurchaseOrder(PurchaseOrderMessage message) {
    try {
        // 1. 检查幂等
        if (idempotentService.tryLock(message.getRequestId(), "ERP_PURCHASE")) {
            log.info("Duplicate message ignored: requestId={}", message.getRequestId());
            return;
        }
        
        // 2. 业务处理
        warehouseService.processPurchaseOrder(message);
        
        // 3. 发送回执
        erpService.sendReceipt(message);
        
        // 4. 释放锁
        idempotentService.unlock(message.getRequestId());
        
    } catch (Exception e) {
        log.error("Process purchase order failed: requestId={}", message.getRequestId(), e);
        throw e; // 触发重试
    }
}
```
