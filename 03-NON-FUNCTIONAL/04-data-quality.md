## 一、数据质量标准

### 1.1 数据完整性

| 数据类型 | 完整性要求 | 检查规则 |
|----------|------------|----------|
| 主键 | 100% | 不允许为空/重复 |
| 外键 | 100% | 必须引用有效主键 |
| 必填字段 | 100% | 不允许为空 |
| 枚举字段 | 100% | 必须在枚举范围内 |
| 日期字段 | 100% | 格式正确，范围合理 |

### 1.2 数据准确性

| 数据类型 | 准确性要求 | 检查规则 |
|----------|------------|----------|
| 库存数量 | ≥99.9% | 实时校验 |
| 批次号 | 100% | 唯一性校验 |
| 物料编码 | 100% | 与ERP同步 |
| 库位编码 | 100% | 层级唯一 |
| 计量单位 | 100% | 标准单位 |

### 1.3 数据一致性

| 数据类型 | 一致性要求 | 检查规则 |
|----------|------------|----------|
| 库存账 | 实时一致 | 库存=入库-出库 |
| 库位占用 | 实时一致 | 占用=实际存放 |
| 单据状态 | 流程一致 | 状态机流转 |
| ERP同步 | T+1同步 | 数据一致 |

## 二、数据校验规则

### 2.1 仓库数据校验

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 仓库编码 | 非空、唯一、长度≤20 | 仓库编码不能为空/已存在/过长 |
| 仓库名称 | 非空、长度≤100 | 仓库名称不能为空/过长 |
| 仓库类型 | 枚举值 | 仓库类型不正确 |
| 温区 | 枚举值 | 温区类型不正确 |

### 2.2 库位数据校验

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 库位编码 | 非空、唯一、格式[A-01-01] | 库位编码不正确 |
| 上级库位 | 存在、层级匹配 | 上级库位不存在 |
| 库位类型 | 枚举值 | 库位类型不正确 |
| 存储容量 | 正整数、最大值限制 | 存储容量不正确 |

### 2.3 物料数据校验

| 字段 | 校验规则 | 错误提示 |
|------|----------|----------|
| 物料编码 | 非空、ERP同步 | 物料编码不能为空 |
| 批次号 | 非空、唯一 | 批次号不能为空/已存在 |
| 失效日期 | > 生产日期、≤10年 | 失效日期不合理 |
| 库存数量 | ≥0 | 库存数量不能为负 |

### 2.4 库存操作校验

| 校验项 | 校验规则 | 错误提示 |
|--------|----------|----------|
| 库存充足 | 出库数量≤可用库存 | 库存不足 |
| 库位可用 | 库位未占用 | 库位已被占用 |
| 批次匹配 | 先进先出 | 请按批次出库 |
| 效期检查 | 出库效期≥当前日期 | 物料已过期 |

## 三、数据追溯

### 3.1 追溯链

```
物料编码 + 批次号
    │
    ├── 生产入库（WMS0090）
    │       └── 采购订单 → 供应商 → 生产批次
    │
    ├── 库位分配
    │       └── 库位变更记录
    │
    ├── 出库操作（WMS0140）
    │       └── 销售订单 → 客户
    │
    ├── 质检记录（WMS0220）
    │       └── 质检结果 → 放行/不合格
    │
    └── 库存调整（WMS0190）
            └── 调整原因 → 审批记录
```

|--------|----------|------|

| 更新人 | updated_by | 谁更新 |
| 更新时间 | updated_at | 何时更新 |
| 业务来源 | source_type | 来源系统/单据 |
| 业务单号 | source_no | 来源单号 |

## 四、数据清理

### 4.1 清理策略

| 数据类型 | 保留时间 | 清理方式 |
|----------|----------|----------|
| 操作日志 | 1年 | 定期归档 |
| 登录日志 | 6个月 | 定期清理 |
| 接口日志 | 3个月 | 定期清理 |
| 临时数据 | 7天 | 自动清理 |
| 历史单据 | 永久 | 不清理 |

### 4.2 清理规则

```sql
-- 清理3个月前的接口日志
DELETE FROM sys_api_log WHERE create_time < DATE_SUB(NOW(), INTERVAL 3 MONTH);

-- 清理7天前的临时文件
DELETE FROM sys_file WHERE is_temp = 1 AND create_time < DATE_SUB(NOW(), INTERVAL 7 DAY);
```

## 五、数据监控

### 5.1 数据质量检查

```sql
-- 检查库存为负
SELECT COUNT(*) FROM wms_inventory WHERE quantity < 0;

-- 检查批次重复
SELECT batch_no, COUNT(*) FROM wms_batch GROUP BY batch_no HAVING COUNT(*) > 1;

-- 检查库位占用不符
SELECT * FROM wms_location WHERE occupied_quantity != 
    (SELECT SUM(quantity) FROM wms_inventory WHERE location_id = wms_location.id);
```

### 5.2 数据异常告警

| 异常类型 | 告警条件 | 告警方式 |

| 同步失败 | 连续失败3次 | 短信+邮件 |

## 六、数据导入规范

### 6.1 Excel导入模板

| 要求项 | 说明 |
|--------|------|
| 文件格式 | .xlsx / .xls |
| 文件大小 | ≤10MB |
| 编码格式 | UTF-8 |
| 必填项 | 黄色标记 |
| 示例数据 | 第一行示例 |

### 6.2 导入校验

| 校验项 | 处理方式 |
|----------|----------|
| 数据格式错误 | 提示具体行和列 |
| 数据重复 | 提示并跳过/覆盖 |
| 必填项为空 | 提示并跳过 |
| 关联数据不存在 | 提示并跳过 |
| 超长数据 | 自动截断或报错 |

### 6.3 导入示例

```java
public ImportResult importWarehouses(MultipartFile file) {
    List<WarehouseImportDTO> list = ExcelUtil.readExcel(file, WarehouseImportDTO.class);
    List<String> errors = new ArrayList<>();
    
    for (int i = 0; i < list.size(); i++) {
        WarehouseImportDTO dto = list.get(i);
        // 1. 校验数据
        if (StringUtils.isEmpty(dto.getWarehouseCode())) {
            errors.add(String.format("第%d行：仓库编码不能为空", i + 2));
            continue;
        }
        // 2. 检查重复
        if (warehouseMapper.existsByCode(dto.getWarehouseCode())) {
            errors.add(String.format("第%d行：仓库编码已存在", i + 2));
            continue;
        }
        // 3. 保存数据
        warehouseService.save(dto);
    }
    
    return new ImportResult(list.size() - errors.size(), errors);
}
```
