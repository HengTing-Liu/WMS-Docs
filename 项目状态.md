**任务描述：** 为 sys_column_meta 增加 4 个扩展字段

```sql
ALTER TABLE sys_column_meta
ADD COLUMN col_span INT DEFAULT 6 COMMENT '表单分组占比权重（1-24）';

#### BE-004：动态 CRUD 接口开发（完成）
**任务描述：** 实现一套通用的动态 CRUD 接口，支持任意表的增删改查

**接口设计：**
```
GET    /api/wms/crud/{tableCode}/list        — 分页列表+高级查询+字段过滤+排序

POST   /api/wms/crud/{tableCode}/save        — 新增保存
PUT    /api/wms/crud/{tableCode}/{id}        — 更新修改
DELETE /api/wms/crud/{tableCode}/{id}        — 删除
POST   /api/wms/crud/{tableCode}/toggle/{id}  — 启禁用切换

#### FE-003：WmsSearchBar 组件扩展（待开发）

**组件功能：** WmsSearchBar 是 WMS 的通用查询栏组件，支持动态配置字段类型

**字段类型映射表：**

| Schema字段类型 | 组件 | 说明 |
|------|------|------|

### 2. sys_column_meta 表结构（BE-002 扩展）

| 字段 | 类型 | 说明 |
|------|------|------|
| col_span | INT | 表单分组占比权重（1-24） |
| section_key | VARCHAR(50) | 分组 Key，对应表单 Section |

### 3. sys_table_operation 表结构参考

**字段清单：** id, operation_code/name/type, button_type, icon, permission, position, sort_order, status, remark, menu_id

### 4. sys_dict / sys_dict_data 表结构参考

**字段清单：** SysDictTypeMapper + SysDictDataMapper，参考 MetaServiceImpl.getColumnSchema() 方法，字典数据通过 DictOptionVO 返回

### 5. 前端路由与后端接口对照

| 前端组件 | 路由 | 说明 |
|------|------|------|
| CrudController | `/api/crud/{tableCode}` | GET list/detail/save/put/delete/checkUnique |
| CrudService + Impl | 通用 CRUD 服务 | 通用 SQL + 动态查询 + 字段过滤 |
| MetaService + Impl | 元数据 Schema 查询 | 按 ColumnMeta 字段 + 字典 + Schema |
| DynamicMapper + XML | 动态 SQL | 按 table_code 动态生成 SQL |
| SqlInjectionValidator | SQL 注入校验 | 白名单校验 + 字段名校验 |
| MetaCacheService | 元数据缓存服务 | `MetaCacheEvent` 缓存刷新 |
| CrudPermissionUtil | 数据权限注入 | `injectDataScope()` 数据范围过滤 |
