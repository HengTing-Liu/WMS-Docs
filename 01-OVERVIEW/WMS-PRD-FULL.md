## 一、产品概述

### 1.1 项目背景

公司自研核心系统（SCM系统）承载了公司核心的业务流程。随着公司业务的快速发展，这个系统现在面临着严峻的挑战：

1. **缺乏统一架构设计，调整困难**：由于早期缺乏统一的架构设计，现在想做一些功能调整或优化变得"非常麻烦"。这导致业务部门的一些合理需求无法被快速响应，新功能上线周期长，影响了业务创新速度。

2. **系统性能瓶颈**：系统变得"越用越卡"，响应缓慢，尤其是在处理复杂操作或数据量大时，大家可能需要等待很长时间，这无疑消耗了大家宝贵的工作时间。

3. **功能耦合，体验割裂**：关键业务环节对实时接口存在强依赖，导致用户操作流程被迫中断，体验不连贯，整体效率低下。

4. **库存管理严谨性不足**：库存计算经常发生计算重复或者不准的情况。

5. **同步接口复杂，需要人工维护**。

### 1.2 项目目标

本WMS（Warehouse Management System）系统聚焦仓储全流程管理，涵盖库存查询、入库、出库、库存调整、质控、提货单、出库单、中转仓、外贸发货、运单配送十大核心模块，支持物料管理、库位分配、质检追溯、物流联动等关键业务场景，实现仓储操作标准化、数据可视化、流程自动化，同步对接ERP、LIMS、OMS等外部系统，确保数据一致性与业务连贯性。

**具体目标**：

1. 所有库存管理相关的功能都在此系统实现；
2. 实现仓库、库区、货位的层级化管理，建立标准化仓储结构体系；
3. 打通入库、出库、盘点、调拨等核心业务流程，确保作业规范与数据实时同步；
4. 支持物料批次、效期、UDI编码管理，实现全生命周期追溯；
5. 提升库存周转率，减少呆滞物料积压，降低资金占用成本；
6. 建立与ERP、LIMS、OA等系统的接口集成，实现数据互联互通；
7. 提供数据统计与分析功能，为仓储决策提供数据支撑。

### 1.3 项目范围

#### V1.0（第一期）

1. 仓储基础数据管理（仓库、库区、货位、物料信息维护）；
2. 核心业务流程（采购入库、销售出库、退货处理、库存调拨、周期盘点）及多仓库协同管理与跨区域调拨；
3. 库存管理（实时库存监控、批次管理、效期预警、UDI关联）；
4. 基础报表（库存台账、出入库明细、盘点差异报表）；
5. 系统管理（用户、角色、权限、日志管理）；
6. 与ERP系统基础数据同步接口。

#### V2.0（第二期）

1. 高级功能（波次拣选、ABC分类管理、库位优化）；
2. 与LIMS系统检测数据对接；
3. 移动端作业（扫码出入库、移动盘点）；
4. 高级分析报表（库存周转率分析、物料ABC分析、异常预警报表）。

## 二、术语表

| 术语 | 解释 |
|------|------|
| WMS | 仓库管理系统（Warehouse Management System），用于仓储业务全流程数字化管理的系统 |
| UDI | 医疗器械唯一标识（Unique Device Identification），医疗器械的"电子身份证"，用于全生命周期追溯 |
| 批次 | 物料生产或入库时形成的批次编号，用于区分不同生产周期或入库批次的物料 |
| 货位 | 仓库内最小存储单元，采用编码规则（如A-01-02）唯一标识 |
| 库区 | 仓库内按功能划分的区域（如常温区、冷藏区、待检区、次品区） |
| 呆滞物料 | 长期未领用或使用频率极低的物料（通常定义为超过6个月无异动） |
| QC | 质量控制（Quality Control） |
| QA | 质量保证（Quality Assurance） |
| CRO | 合同研究组织（Contract Research Organization） |
| LIMS | 实验室信息管理系统（Laboratory Information Management System） |
| ERP | 企业资源计划（Enterprise Resource Planning） |

## 三、功能需求

### 3.1 基础设置

#### 3.1.1 仓库档案（WMS0010）

**功能说明**：管理仓库基本信息、温区、质检分区等。

**主要功能**：
- 查询仓库列表（按仓库编码、名称、状态等条件筛选）
- 新建仓库档案
- 修改仓库档案
- 启用/停用仓库

**字段定义**：

| 字段名 | 组件 | 必填 | 唯一 | 说明 |
|--------|------|------|------|------|
| 仓库ID | 数字 | 是 | 是 | 系统字段 |
| 仓库编码 | 文本 | 是 | 是 | 用于系统内部识别 |
| 仓库名称 | 文本 | 是 | 否 | 仓库显示名称 |
| 仓库类型 | 下拉 | 是 | 否 | 销售仓/隔离仓/前置仓/中转仓/退货仓 |
| 所属公司 | 文本 | 是 | 否 | 仓库所属公司 |
| 温区 | 下拉 | 是 | 否 | 常温/冷藏/-18℃冷冻/-40℃超低温 |
| 质检分区 | 下拉 | 否 | 否 | 合格区/待检区/隔离区/不合格区 |
| 负责人 | 选择 | 否 | 否 | 仓库管理员 |
| 备注 | 多行文本 | 否 | 否 | 备注信息 |
| 状态 | 开关 | 是 | 否 | 启用/停用 |

#### 3.1.2 库位档案（WMS0020）

**功能说明**：管理仓库内的库位层级结构（存储类型 → 存储分区 → 存储容器 → 存储孔位）。

**主要功能**：
- 查看库位层级树形结构
- 新建存储类型（冰箱/货架/地堆/托盘）
- 新建存储分区（按层级配置）
- 新建存储容器（独占/共享模式）
- 批量生成存储孔位
- 修改库位信息
- 删除库位（级联删除下级）
- 分配仓库档案

**层级结构**：

```
仓库
└── 存储类型（冰箱/货架/地堆/托盘）
    └── 存储分区（层/架/行/列/格）
        └── 存储容器（盒/箱/笼/抽/屉）
            └── 存储孔位（A01, A02...）
```

| 出口名称 | 用于出口申报的名称 |
| 品牌 | 物料品牌 |
| 货号 | 货号 |
| 产品类别 | 产品分类 |
| 规格 | 物料规格 |
| 计量单位 | 计量单位 |
| 包装规格 | 包装规格 |
| 是否必检 | 是否需要质检 |
| UDI-DI编码 | 医疗器械唯一标识 |

| 批次字段 | 说明 |
|----------|------|
| 批次号 | 系统生成的批次号 |
| 供应商批号 | 供应商提供的批次号 |
| 纯化编号 | 纯化编号 |

## 五、页面分类与开发策略

### 5.1 页面分类

| 类别 | 特征 | 开发方式 | 数量 |
|------|------|----------|------|
| A类 | 纯档案页面（增删改查、启用停用） | 低代码 | ~12个 |
| B类 | 简单单据（单表 + 状态机） | 低代码 + 业务扩展 | ~7个 |
| C类 | 复杂业务（多行明细、多阶段流程） | 定制Vue组件 | ~23个 |

### 5.2 开发优先级

**第一优先级（V1.0核心）**：
- 基础设置（仓库、库位、物料、基础数据）
- 库存查询
- 入库操作
- 出库操作
- 库存调整
- 质量控制
- 系统管理

**第二优先级（V1.0扩展）**：
- 提货单管理
- 出库单管理

**第三优先级（V2.0）**：
- 中转仓管理
- 外贸发货
- 运单配送
