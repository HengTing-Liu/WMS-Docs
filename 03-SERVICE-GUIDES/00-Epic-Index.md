## 三、开发顺序建议（调整后）

### Phase 0：基础设施（P0，最优先）

```
Epic-11 低代码配置管理 ──► 前端低代码组件
          │
          └──► 为后续所有页面开发提供基础设施
```

**原因**：Epic-11 是所有前端页面的基础设施，后续 Epic-01~10 的页面开发都需要依赖低代码组件。

### Phase 1：核心基础设施（P0）

```
Epic-11 ──► Epic-01 基础数据管理 ──► Epic-10 系统管理 ──► Epic-02 库存查询
                    │                        │                       │
                    └────────────────────────┴───────────────────────┘
                                          │
                                    数据库表 + 权限框架 + 查询基础
```

### Phase 2：核心业务（P0）

```
Epic-03 入库管理 ──► Epic-04 出库管理 ──► Epic-06 质量控制
         │                        │
         └──► 库存一致性 + 状态机
```

### Phase 3：扩展业务（P1）

```
Epic-05 库存调整 ──► Epic-07 提货单管理
         │                    │
         └────────────────────┘
               盘点 + 提货
```

### Phase 4：物流与出库（P1~P2）

```
Epic-08 出库单管理 ──► Epic-09 物流发货
          │                    │
          └────────────────────┘
                中转仓 + 配送
```

### Phase 5：数据支撑（P1）

```
Epic-12 报表与统计 ──► Epic-13 预警通知管理
         │                         │
         └──► 依赖所有业务Epic完成后开发
```

## 四、跨Epic依赖关系

| 依赖关系 | 前置Epic | 后置Epic | 说明 |
|---------|----------|----------|------|
| 低代码→页面 | Epic-11 | Epic-01~10 | 低代码是页面开发基础 |
| 入库→库存 | Epic-03 | Epic-02 | 入库后需更新库存查询 |
| 出库→库存 | Epic-04 | Epic-02 | 出库后需更新库存查询 |
| 盘点→库存 | Epic-05 | Epic-02 | 盘点调整影响库存 |
| 提货→出库 | Epic-07 | Epic-08 | 提货单生成出库单 |
| 出库→物流 | Epic-08 | Epic-09 | 出库单进入物流流程 |
| 质检→入库 | Epic-06 | Epic-03 | 质检影响入库状态 |
| 业务→报表 | Epic-01~09 | Epic-12 | 报表依赖业务数据 |
| 业务→预警 | Epic-01~09 | Epic-13 | 预警依赖业务数据 |
