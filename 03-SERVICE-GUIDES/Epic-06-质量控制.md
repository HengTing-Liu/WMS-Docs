## 一、Epic 概述

### 1.1 业务背景

质量控制是医疗器械仓储的核心合规要求，覆盖质检标准管理、请验申请、质检执行、放行决策等全流程。

### 1.2 包含模块

| 模块编号 | 模块名称 | Story数 | 开发顺序 |
|---------|---------|---------|---------|
| WMS0215 | 质检标准 | 4 | 1 |
| WMS0220 | 质量放行 | 5 | 2 |

---

## 二、模块 Story 拆分

### 2.1 WMS0215 质检标准

| Story编号 | Story名称 | 验收标准 | 依赖关系 |
|---------|---------|---------|---------|
| S-0215-01 | 质检标准列表 | 按分类、状态筛选 | - |
| S-0215-02 | 质检标准新增 | 填写标准信息、项目配置 | S-0215-01 |
| S-0215-03 | 质检标准编辑 | 修改标准信息、项目 | S-0215-01 |
| S-0215-04 | 质检标准删除 | 逻辑删除、关联检查 | S-0215-01 |

**关键字段**：
- 标准编码、标准名称、分类（医疗器械/药品/体外诊断）
- 检验项目（外观、性状、含量、效期等）
- 判定标准、合格范围

**数据库表**：`qc_standard`

**权限标识**：`wms:qc:standard:query`、`wms:qc:standard:add`、`wms:qc:standard:edit`

---

### 2.2 WMS0220 质量放行

| Story编号 | Story名称 | 验收标准 | 依赖关系 |
|---------|---------|---------|---------|
| S-0220-01 | 请验单列表 | 按状态、物料筛选 | - |
| S-0220-02 | 请验单新建 | 选择物料、批次、填写样品信息 | S-0220-01 |
| S-0220-03 | 请验执行 | LIMS同步质检结果 | S-0220-02 |
| S-0220-04 | 放行决策 | 合格/不合格/让步放行 | S-0220-03 |
| S-0220-05 | 放行ERP同步 | 同步结果至ERP | S-0220-04 |

**状态流转**：`PENDING → IN_PROGRESS → QUALIFIED/UNQUALIFIED/CONDITIONAL`

**关键字段**：
- 请验单号、质检标准、物料、批次、样品数量
- 检验结果、判定结论、放行人、放行时间

**数据库表**：`qc_record`

**权限标识**：`wms:qc:record:query`、`wms:qc:record:add`、`wms:qc:record:approve`

**核心约束**：
- 请验期间冻结库存（可用库存 = 账面 - 冻结）
- 让步放行需上级审批
- 不合格物料需隔离处置

## 三、技术实现要点

### 3.1 请验冻结流程

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                            请验冻结流程                                      │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  创建请验单 ──▶ 提交 ──▶ 冻结库存 ──▶ 送检 ──▶ 等待LIMS结果               │
│                      │           │                 │                        │
│                      │           ▼                 ▼                        │
│                      │    库存冻结标志        LIMS同步                      │
│                      │    可用库存减少        质检进行中                    │
│                      │                          │                          │
│                      │                          ▼                          │
│                      │                    质检完成                          │
│                      │                          │                          │
│                      │         ┌───────────────┼───────────────┐            │
│                      │         ▼               ▼               ▼            │
│                      │     合格             不合格         让步放行         │
│                      │         │               │               │            │
│                      │         ▼               ▼               ▼            │
│                      │    解冻库存         维持冻结         部分解冻         │
│                      │    继续流通        隔离处置         审批后流通       │
│                      │                                                             │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 LIMS 同步

| 同步方向 | 触发时机 | 技术方案 |
|---------|---------|---------|
| WMS → LIMS | 请验单提交 | MQ 异步推送 |
| LIMS → WMS | 质检完成 | Webhook / 定时拉取 |

## 四、测试要点

### 4.1 业务测试

| 测试场景 | 预期结果 |
|---------|---------|
| 请验申请 | 冻结库存、更新质检状态 |
| 取消请验 | 解冻库存 |
| 合格放行 | 解冻库存、物料可出库 |
| 不合格 | 维持冻结、触发隔离 |
| 让步放行 | 部分解冻、需审批 |

### 4.2 LIMS 接口测试

| 测试场景 | 预期结果 |
|---------|---------|
| 同步成功 | 更新质检结果、触发放行流程 |
| 同步失败 | 记录日志、触发重试 |
| 重复同步 | 幂等处理、不重复处理 |
